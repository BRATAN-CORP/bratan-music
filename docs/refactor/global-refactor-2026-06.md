## Что осознанно НЕ тронуто и почему

- **Player.tsx ↔ FullscreenPlayer.tsx клоны (~120 строк)** — это две
  независимые поверхности одного дизайна; унификация дала бы связность
  хуже дублирования и риск визуальных регрессий (hard constraint «не
  ломать плеер/дизайн»).
- **worker/ исходники не удалены** — каталог уже не в проде, но (а) от
  него зависит `worker/init-db` (mount у postgres), (б) docs/structure
  ссылается на него как на исторический контракт портирования. Удаление
  — отдельный PR после ручной проверки Go-бэка на проде.
- **Дублирование bot ↔ api-go ↔ worker слоёв** — межъязыковое (TS
  легаси vs Go), унификация невозможна по определению; Go-порт и есть
  её устранение.
- **D1-миграции, схема БД, формат конфигов, публичные API** — вне скоупа.
- **`catch { /* ignore */ }` в аудио-движке (~165 мест)** — почти все
  задокументированы и осознанны (Web Audio API кидает на легитимных
  путях); глобальная замена рискует поведением.
- **TidalService.ts (1280 строк, legacy worker)** — режется только вместе
  с удалением worker/.
- **`useLikeTrack`/`useUnlikeTrack`** — похожи на album/artist-хуки, но
  отличаются ключами кэша, набором инвалидаций и offline-sync kind;
  унификация добавила бы 5 параметров ради 30 строк — связность хуже
  дублирования.
