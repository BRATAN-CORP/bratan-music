# BRATAN MUSIC — База знаний

Это локальный Obsidian-style vault проекта. Дерево markdown-файлов,
которые читают и обновляют ИИ-агенты (Devin / Claude / Codex). Можно
открыть всю папку `docs/` как vault в Obsidian — wiki-ссылки `[[…]]`,
графы связей и плагины работают из коробки.

## Содержание

- [[context/index|context]] — глобальная информация о проекте: что такое
  BRATAN MUSIC, какие проблемы решает, стек, бизнес-правила, лицензия,
  ссылки на внешние ТЗ и заметки.
- [[structure/index|structure]] — карта кода. Структурно повторяет
  репозиторий и описывает, **что**, **где** и **зачем** реализовано.
  Это основной раздел, с которым работают агенты при решении задач.
- [[daily-changes/index|daily-changes]] — журнал изменений в коде и в
  базе знаний. Сюда пишет каждый агент после своей задачи. Тут же
  лежат roadmap'ы крупных рефакторингов в виде trackerов.

## Конвенции для агентов

### При получении задачи

1. Прочитай `docs/README.md` (этот файл).
2. Зайди в [[structure/index|structure]] и найди подсистему, которая
   относится к задаче (frontend / worker / telegram-bot / data).
3. По ссылкам спустись только в тот раздел, который нужен. Не читай
   весь код — это и есть смысл базы знаний.
4. Если в задаче упоминается продукт ("плеер", "комнаты",
   "AI-плейлисты") — открывай [[context/index|context]] для бизнес-
   контекста.

### После выполнения задачи

1. Если изменилась структура кода (новый файл/модуль/удаление) —
   обнови соответствующий index в `docs/structure/`. Файлы > 300
   строк дроби на подфайлы и заменяй ссылками.
2. Создай / дополни запись в `docs/daily-changes/YYYY-MM-DD.md`
   (см. [[daily-changes/index|daily-changes]]).
3. Если ведёшь длительный рефакторинг — отметь прогресс в
   `docs/refactor/<имя>.md` (tracker по подзадачам).

### Размер файлов

Не больше ~300 строк на index. Когда раздел разрастается —
выноси в подпапку с собственным index.md и линкуй. Это критично:
агенты режут контекст по файлам, и крупный index перестаёт
влезать в одно чтение.

### Wiki-ссылки

Используем формат `[[путь/index|подпись]]` для всего, кроме
внешних ссылок (на код / GitHub / документацию — обычные
markdown-ссылки). Это позволяет Obsidian рисовать граф и при
переименовании каскадно обновлять линки.
