# Плавность анимаций на iOS — perf/ios-animations

**Когда:** 2026-06-10 (поздний вечер) UTC
**Кто:** Viktor (AI)
**Branch:** `perf/ios-animations`
**Базис:** `main` @ `e8cd8b9`

## Контекст

Пользователь: «на пк вроде +- норм, а на иосе на мобиле прям все рвато
как-то. свайпы кривые». Профилирование кода нашло несколько источников
дропнутых кадров, специфичных для mobile WebKit.

## Root causes и фиксы

1. **`useBassPulse` ре-рендерил весь FullscreenPlayer 60 fps** —
   хук писал `setState` в rAF-цикле на каждый кадр, пока играет музыка
   в фуллскрине. Дерево фуллскрин-плеера огромное (~1500 строк JSX) —
   это главный источник «рвато». Теперь хук пишет в **MotionValues**
   (тот же паттерн, что `usePlaybackVisuals`): React вообще не
   ре-рендерится от аудио-сигнала, `pulse`/`flash` деривируются через
   `useTransform`. (`useAudioPlayer.ts` — хирургическая правка одной
   экспортируемой функции, `FullscreenPlayer.tsx` — потребитель.)

2. **Гало анимировало `filter: blur(56→120px)` каждый кадр** — mobile
   WebKit пере-растеризует огромный блюр-слой на каждое изменение
   радиуса. Блюр теперь константный (`blur(88px) saturate(1.5)`),
   дыхание несут только `scale` + `opacity` (чисто композиторные).
   Сглаживание уже есть в энвелоупах хука, визуально пульс сохранён.

3. **SwipeTrackStrip: хитч на первом кадре свайпа** — edge-fade маска
   включалась через `setDragging(true)` в `onDragStart`: React-рендер +
   растеризация WebKit-маски ровно в момент начала жеста. Маска теперь
   статически висит на ghost-колонках (центр всегда был непрозрачным),
   состояние `dragging` удалено целиком. Плюс `willChange: transform`
   на стрипе — он живёт внутри `liquid-glass` (backdrop-filter), без
   пре-промоута слой создавался на первом кадре драга.

4. **`willChange: transform`** также на обложке фуллскрина (drag='x'
   над двумя блюр-слоями) и на sheet-обёртке свайпа-вниз.

5. **Backdrop-blur дешевле на тач-устройствах** — `globals.scss`:
   `@media (hover: none) and (pointer: coarse)` снижает радиус
   `liquid-glass` 22→14px и `.glass` 20→12px. На телефонной диагонали
   разница не видна, а стоимость композитинга падает заметно (док с
   мини-плеером висит над скроллящимся контентом). Десктоп не тронут.

- `tsc --noEmit`, `bun run lint`, `bun run build` — чисто.
- `useAudioPlayer.ts` — только тело `useBassPulse` (state → MotionValue),

## Не вошло / follow-up

- `transition-all` встречается ~31 раз по коду — можно точечно заменить
  на `transition-colors`/`transition-transform`, но это косметика
  по сравнению с пп. 1–2.
- Ambience-кроссфейд при смене трека держит два полноэкранных
  `blur-3xl` слоя 0.85 с — одномоментный спайк, можно перейти на
  низкоразмерную обложку для бэкдропа, если всё ещё будет заметно.
