   wiki-ссылки, плагины работают.

### Что важно знать следующему агенту
- **Размер index-файлов держим до ~300 строк.** Если раздел разрастается
  — выноси в подпапку с собственным `index.md`.
- **Wiki-ссылки** — формат `[[путь/index|подпись]]` для внутренних
  переходов. Для внешних (GitHub, код, MDN) — обычные markdown-ссылки.
- **Лог обязателен.** После каждой задачи добавляй запись в
  `docs/daily-changes/YYYY-MM-DD.md` по формату из
  [[index|daily-changes/index.md]].
- **Tracker'ы.** Для длительных рефакторингов заводи
  `docs/refactor/<имя>.md` (по запросу пользователя — первый такой
  файл должен быть создан перед началом UI/UX-рефакторинга).
- **Не дублируем README.** Если что-то есть в `README.md` — линкуем,
  не копируем. База знаний — про **где найти** и **зачем**, а не
  про маркетинг.

### Что изменилось
- Новый файл `src/lib/dailyVariant.ts` — единый источник правды для:
  - `DAILY_VARIANT_THEME` — `hue` / `labelKey` / `nameKey` / `descKey` /
    `icon` для всех трёх вариантов «Плейлист дня» (`familiar` /
    `discover` / `mood`).
  - `dailyTrackUnitKey(count)` — i18n-ключ для падежной формы
    счётчика треков.
- `src/app/home/page.tsx` и `src/app/daily/page.tsx` теперь импортируют
  из `@/lib/dailyVariant` вместо локальных копий. В `home/page.tsx`
  заодно ушёл неиспользуемый `Heart` из импортов lucide-react.

### Почему
Идентичный 25-строчный `VARIANT_THEME` и почти идентичный
`dailyTrackUnitKey` жили в двух файлах. Любая правка hue / описания
требовала держать обе копии в синхроне. Запрос пользователя:
«меньше хардкода и повтора кода … унифицировать код».
