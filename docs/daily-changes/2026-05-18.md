### Корневые причины

**Цензура внутри альбомов**. PR #457 swap'ил clean→explicit ТОЛЬКО
для треков (`swapInExplicitTwins` в search/artistTopTracks/
playlistTracks). Альбомы получали лишь same-response prefer-explicit
dedupe (`preferExplicitAlbums`) — этот слой работает только когда
Tidal в одном ответе отдал и clean и explicit вариант. Когда Tidal
возвращал ТОЛЬКО clean album_id (типичный случай для пул-аккаунта с
включённым explicit-фильтром), search-ответ был чистым "clean-only",
dedupe ничего не делал, юзер кликал на album, `getAlbum(cleanId)`
честно отдавал clean-tracklist (контракт by-id retrieval — мы не
знали что есть explicit twin). Юзер сам подтвердил: "если беру id
explicit-альбома напрямую из tidal — uncensored. айдишники при этом
различные".

**Пропадание E на player surfaces**. Поле `explicit` дропалось в
4+ setTrack call-сайтах:

- `src/hooks/usePlaybackSync.ts :: toPlayable()` — тип
  `PlayableTrack` был `Pick<Track,'id'|'title'|'artist'|'duration'>
  & Partial<Pick<Track,'artistId'|'albumId'|'coverUrl'|'coverVideoUrl'>>`
  — БЕЗ `explicit`, `artists`, `source`. Этот хук используется во
  всех TrackItem-кнопках через `useTrackPlayback` /
  `useCollectionPlayback` (search, library, downloaded, daily,
  explore, …).
- `src/components/features/ExploreModules.tsx` — два inline
  `setTrack({id,title,artist,artistId,artists,coverUrl,coverVideoUrl,
  duration})` объекта (row-level и playlist-cover-button), оба без
  `explicit`.
- `src/app/explore/list.tsx :: ListView` — тот же паттерн.
- `src/app/explore/playlist.tsx :: handlePlayTrack` — тот же
  паттерн.

В результате currentTrack.explicit становился undefined → лоджер
плеев (`usePlayHistoryLogger`) писал 0 в `play_history.explicit` →
recent strip отрисовывался без бэйджа. Mini-player и FullscreenPlayer
тоже не показывали E (они читают `currentTrack.explicit` напрямую).
А TrackItem на странице альбома/плейлиста показывал E корректно
потому что брал поле прямо из source-snapshot'а.

**Кривой бэйдж**. PR #454 и PR #457 пытались чинить optical centering
через `transform: translateY(-Xpx)` на инлайновом span'е с буквой
"E". Это font-метрический хак: точное смещение зависело от шрифта
(Inter / Instrument Serif / system fallback), DPR, размера. Юзер
жаловался дважды: "сама e расположена не по центру квадратика" и
после PR #457 — ровно то же самое.

### Что сделано

#### (A) Worker — album-level twin lookup + getAlbum redirect

В `worker/src/services/tidal/TidalService.ts` три новых private
метода:

- **`swapInExplicitAlbumTwins(items: Album[]): Promise<Album[]>`** —
  альбомный аналог `swapInExplicitTwins`. Идентифицирует clean
  альбомы (explicit !== true) с known `artistId`, для каждого
  asynchronously резолвит explicit twin, substitute'ит на тот же
  индекс. Order-preserving, concurrency=4, silent fall-through на
  ошибках.
- **`resolveExplicitAlbumTwin(clean: Album): Promise<Album | null>`** —
  KV-memo `tidal-explicit-album-twin:<cleanId>` 30d (положительный
  результат = explicitId, отрицательный = `__none__`). На miss —
  Tidal `search(query, 'ALBUMS', 20)` по `(title, artistName)`,
  matching по `(artistId, normaliseAlbumTitle(title), explicit===true)`,
  hydrate'ит первое совпадение через `getAlbum`+`getAlbumTracks`
  чтобы вернуть полный payload с tracks.
- **`resolveExplicitAlbumIdRedirect(id: string): Promise<string>`** —
  быстрый KV path для hot redirect-навигации. Сначала читает
  `tidal-explicit-album-twin:<id>` (negative cache → возвращает
  оригинал; positive → возвращает twin id). На miss — `getAlbum(id)`,
  если уже explicit — пишет `__none__` в KV и возвращает оригинал;
  иначе летит в `resolveExplicitAlbumTwin`.

Применено в:
- `TidalService.search()` — albums (параллельно с tracks через
  `Promise.all([swapInExplicitTwins, swapInExplicitAlbumTwins])`).
- `TidalService.getArtistAlbums()`.
- `TidalService.getArtistAlbumsAndSingles()` — оба bucket'а.
- `TidalService.getArtistReleases()` — после dedupe+sort.
- `TidalService.getArtistReleasesPage()` — paginated "see all".
- `TidalService.getExploreList(albums)` — explore "see all".
- `TidalService.getExplorePage()` — параллельно по модулям.

**Главное изменение — `TidalService.getAlbum()` теперь делает
transparent redirect**: если `resolveExplicitAlbumIdRedirect`
возвращает другой id (= existed explicit twin in KV), фетч идёт по
explicit id, и пользователь получает uncensored tracklist даже по
deep-link / library-bookmark / Telegram-share clean'а. Это
отличается от поведения PR #457 (`getAlbum` оставался by-id-only,
только warn breadcrumb). Юзер явно попросил такое поведение:
"если беру id explicit-альбома напрямую из tidal — uncensored.
айдишники при этом различные (видимо вариации альбомов есть с
цензурой и без)". Audio engine, `TidalWeb.resolveStream`,
`playbackinfopostpaywall` НЕ тронуты.

Hot-path overhead на `getAlbum`: один KV read. На КВ-miss —
дополнительный `/v1/albums/{id}` фетч + (иногда) `/v1/search?type=ALBUMS`
+ KV write. Положительный И отрицательный исход
кэшируются на 30 дней, так что репеты — почти всегда cache hits.

#### (B) Worker — getExplorePage по-модульный swap

После `mapPageModule` цикла, каждый module с
`type === 'tracks' | 'albums'` параллельно через

`swapInExplicitTwins`/`swapInExplicitAlbumTwins` поверх

union в `worker/src/types/music.ts :: ExploreModule` обеспечивает
правильное narrowing на `mod.items`.

#### (D) ExplicitBadge — SVG rewrite

`src/components/features/ExplicitBadge.tsx` полностью переписан:

- 10×10 viewBox.
- Подложка: `<rect x="0" y="0" width="10" height="10" rx="1.6"
  ry="1.6">`.
- Буква "E": единый closed path
  `d="M2.4 2 H7.6 V3.4 H3.4 V4.5 H7 V5.5 H3.4 V6.6 H7.6 V8 H2.4 Z"`
  с тремя горизонтальными перекладинами (top y=2→3.4, middle
  y=4.5→5.5, bottom y=6.6→8) и спайном (x=2.4→3.4). Геометрически
  центрируется в `(5, 5)`: top+bottom-перекладины равноудалены от
  центральной линии (top centre = 2.7, bottom centre = 7.3, ⌀
  идеальный 5.0), middle — точно на ней, спайн равен по толщине
  перекладинам.
- Никаких translateY хаков, никакой font-метрики, никакой
  DPR-зависимости.
- Размеры 12 / 14 / 16 / 18 / 20 / 24 px рендерятся одинаково
  ровно — вся геометрия скейлится через viewBox.
- `tone='light'` сохранён (белая `rgba(255,255,255,0.92)` подложка +
  чёрный `rgba(0,0,0,0.85)` foreground для FullscreenPlayer).
- CSS-custom-properties (`var(--color-text-muted)`,
  `var(--color-bg)`) применяются через inline `style.fill` (не
  через SVG presentation attribute `fill="…"`) — кросс-браузерная
  стабильность (исторически SVG-presentation поддержка
  `var(...)` имела квирки).

#### (E) MediaSession like (bonus)

В `src/components/layout/Player.tsx` новый `useEffect` (после
существующего `liked = currentTrack ? isLiked(currentTrack.id) : false`):

- Регистрирует `setActionHandler` под кандидатными именами
  `togglelike` / `like` / `favorite` / `favourite` / `star`.
  W3C MediaSession-спека формально like не определяет (см.
  https://www.w3.org/TR/mediasession/#mediasessionaction-enum), но
  Chromium и разные платформы (Android, ChromeOS, CarPlay через
  системные обёртки) пробуют разные имена. На iOS PWA Safari /
  Web App это best-effort.
- Каждый `setActionHandler(action, handler)` обёрнут в try/catch
  — браузер либо подхватывает один из вариантов и показывает
  звёздочку в lock screen / Now-Playing widget, либо silently
  no-op. На очистке также try/catch.
- Handler читает live state из
  `usePlayerStore.getState().currentTrack` и вызывает
  `useToggleLike().toggle(...)` — точно тот же путь что и
  кнопка-сердечко в мини-плеере, так что состояние
  "лайкнуто / не лайкнуто" остаётся в синхроне с библиотекой
  пользователя.
- Зависимости эффекта: `[currentTrack?.id, toggle]` — re-register
  при смене трека, чтобы handler closure'ил правильный live
  state.

Audio engine не тронут (новый effect живёт в `Player.tsx`, не в
`useAudioPlayer`).
