## Контекст

Пользователь хочет перенести лайкнутые треки из Яндекс.Музыки в BRATAN
MUSIC. У Яндекс.Музыки нет публичного API, поэтому поток построен через
файловый экспорт: пользователь выгружает библиотеку через
TuneMyMusic/Soundiiz/MusConv («Export to file» → CSV) и загружает файл
на новую страницу «Импорт треков» в библиотеке.

## Что сделано

### Бекенд (`api-go`)

- `POST /api/import/likes` — принимает `{tracks: [{title, artist,
  album?, isrc?, duration?}]}` (до 3000 строк), запускает фоновую
  goroutine, отвечает `{jobId, total}`.
- `GET /api/import/likes/{jobId}` — прогресс: `{total, processed,
  matched, failed[], done}`. Джобы user-scoped, реестр in-memory
  (один инстанс api-go в деплое), TTL 1 час после завершения.
- Матчинг (`matchImportRow`):
  1. **ISRC** — точное совпадение кода записи (один и тот же у трека во
     всех сервисах), без эвристик.
  2. **Метаданные** — нормализованные артист+название (срезаются
     feat-кредиты, скобки, пунктуация) + проверка длительности.
     Скоринг отбрасывает каверы/караоке (чужой артист = 0) и
     ремиксы/лайвы (длительность мимо >15 c = −3). Порог: точное
     название + совпавший артист.
- Совпавшие треки лайкаются напрямую в liked-плейлист с полным
  snapshot (тот же формат, что фронт пишет при ручном лайке),
  дубликаты пропускаются.
- `tidal.TrackRaw` получил поле `ISRC` (приходит в `/v1/search`).
- Пауза 120 мс между поисками — не душим пул Tidal-аккаунтов.
- Тесты: ISRC-матч, метадата-матч, not-found, отсев чужого артиста,
  нормализация (включая кириллицу).

### Фронтенд

- `src/lib/importCsv.ts` — парсер CSV-экспортов: сниффинг разделителя
  (`,`/`;`), маппинг колонок по имени (en/ru варианты Soundiiz,
  TuneMyMusic, MusConv), длительность в `м:сс`/секундах/мс,
  дедупликация строк.
- `src/app/library/import/page.tsx` — страница в стиле uploads-страницы:
  выбор CSV → превью (количество + первые 5 строк) → запуск → поллинг
  прогресса (1.5 c) с прогресс-баром → финальный отчёт со списком
  ненайденных. По завершении инвалидируются `['liked']` и
  `['playlists']`.
- Тайл «Импорт из других сервисов» в библиотеке (рядом с «Мои
  загрузки»), роут `library/import`, i18n-ключи `importLikes.*` +
  `library.importEntry*` в ru/en.

## Осознанные ограничения

- Реестр джобов in-memory: при рестарте api-go активный импорт
  пропадает (страница перестанет получать статус) — пользователь
  просто запускает импорт заново, уже добавленные лайки не дублируются.
- Матчинг консервативный: лучше «не нашли» (трек попадёт в отчёт), чем
  тихо залайканный кавер.
