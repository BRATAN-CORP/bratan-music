# Daily changes

Журнал изменений базы знаний и проекта. Каждый агент после своей задачи
оставляет здесь короткую запись: **что**, **где**, **почему**.
Длительные рефакторинги — отдельным tracker-файлом в `docs/refactor/`.

## Формат записи

Один файл на дату: `YYYY-MM-DD.md`. Внутри — список изменений (один
markdown-section на PR / задачу):

```markdown
## <PR title or task slug> — <PR number or branch>

**Когда:** YYYY-MM-DD HH:MM UTC
**Кто:** <agent / user>
**Branch:** <branch>
**PR:** <link>

### Что изменилось
- bullet 1
- bullet 2

### Почему
короткий контекст / причина / ссылка на task

### Документация в `docs/`
- [[structure/<...>]] — обновил такой-то раздел
- [[context/<...>]] — добавил/изменил
- (если нет) "не менялась"

### Что важно знать следующему агенту
- caveat / гейтинг / то, что не в коде
```

## Каноничный пример

См. [[2026-05-08]] — первая запись (создание самой базы знаний).

## Текущие активные tracker'ы

- `docs/refactor/<имя>.md` — крупные многошаговые рефакторинги.
  Создаётся на старте, обновляется галочками на каждом step'е.
  Назначение: чтобы любой следующий агент мог продолжить, открыв один
  файл.

## Ретенция

Файлы старше ~3 месяцев архивируются в подпапку `archive/YYYY/`,
если тема перестала быть актуальной. Для долгосрочной справки —
информацию переносим в [[../structure/index|structure]] /
[[../context/index|context]] **до** архивирования.
