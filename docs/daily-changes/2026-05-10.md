### Что изменилось

**1. Queue modal — центрирование на любом breakpoint.**
- `src/components/features/QueueDialog.tsx`:

    `<Modal align="center">`. Теперь панель центрируется в viewport на

**2. Queue drag — split touch vs mouse.**
- Drag-to-reorder теперь идёт через `useDragControls()` +
  `Reorder.Item dragListener={false}`, как уже сделано в
  `PlaylistTrackItem`.
- На `Reorder.Item.onPointerDown`: `dragControls.start(e)` вызывается
  только если `e.pointerType === 'mouse'`. Это сохраняет старое
  поведение «драг с любого места» для мыши (на десктопе нет конфликта
  scroll-vs-drag — колесо скроллит, pointer-down таскает строку).
- На handle `<span>` (с `<GripVertical>`) `onPointerDown`:
  `e.preventDefault() + e.stopPropagation() + dragControls.start(e)`.
  preventDefault подавляет нативный text-select / image-drag, а
  stopPropagation предотвращает двойной запуск от bubble-up при mouse.
- Handle получил `touch-none`, чтобы палец на иконке шёл напрямую в
  motion-pipeline; остальная строка наследует `pan-y` от родителя
  (`data-allow-pan-y` на queue-list контейнере), так что touch
  где-нибудь, кроме иконки, скроллит список нативно.
- Добавлен i18n-ключ `queue.dragHandle` (RU «Перетащить трек» / EN
  «Drag track»), `aria-label` + `role="button"` на handle для
  доступности.

**3. Lyrics на <md — замещают обложку, кебаб → крестик.**

    Lyrics-блок: `flex-1 min-h-0 w-full max-w-md overflow-hidden`,
    `data-no-sheet-drag` чтобы не триггерить swipe-down dismiss
    fullscreen-плеера. Это и реализует «замещали обложку и пространство
    сверху чуть» — flex-1 поглощает свободное вертикальное

  - Удалён `<LyricsPanel mode="overlay">` рендер (был

### Что изменилось

**1. Lyrics loader/error/empty — по центру cover-slot.**
`LyricsContent` early-return состояния (`isLoading` / `isError` / `!data?.available`)
получили `w-full` рядом с `flex h-full items-center justify-center`.
Раньше parent cover-slot был `flex` (row), без `flex-1` на child лоадер
получал content-width — `justify-center` центрировал внутри
content-width = «не центрировал». С `w-full` flex-row child тянется
на всю ширину слота → spinner + текст реально по центру.

**2. Blur(0.5px) на неактивных строках лирики убран.**
В `motion.p.animate.filter` теперь всегда `'blur(0px)'`. Полупиксельный
blur на DPR≥2 (мобила) рендерился как видимая «дымка», которая исчезала
во время смены активной строки и наплывала обратно — пользовательская
жалоба «блюр пропадает когда строчка переключается и потом опять
накладывается». На PC DevTools без DPR-эмуляции 0.5px округлялся до 0
→ эффекта не было, отсюда расхождение. Оставлены только opacity + scale
как глубинные cues — то, что и так фактически рисовалось на десктопе.

**3. Лирика расширена вверх через absolute-inner wrapper.**
`FullscreenPlayer` cover-slot keeps OUTER `relative aspect-square w-full
max-w-md` + maxWidth-clamp (фиксированный bounding box в flow → элементы
плеера не двигаются), НО `overflow-hidden` заменён на `overflow-visible`,
а внутри добавлен absolute wrapper:

```
<div className="absolute inset-x-0 bottom-0" style={{ top: '-5rem' }}>
  <LyricsContent ... />
</div>
```

Bottom прижат к нижней грани слота, top уходит на 5rem ВЫШЕ верхней
грани слота → лирика визуально занимает breathing-зону над обложкой.
5rem — экспирически устойчивая величина: phone (~600px viewport) даёт
~7rem breathing → остаётся 2rem gap до X-icon шапки; tablet (~900px)
даёт ~14rem breathing → 7rem gap. Title/progress/transport/volume
строго на тех же пикселях.

**4. Клик по артисту в FullscreenPlayer → плавное сворачивание + переход.**
- `ArtistLinks` получил новый prop `onNavigate?: () => void`. Обработчик
  обоих веток (single-credit `<Link>` + multi-credit map) теперь вызывает
  `handleLinkClick(e)` → `e.stopPropagation()` (если включено) → `onNavigate?.()`.
- `FullscreenPlayer` multi-credit branch передаёт `onNavigate={closeFullscreen}`.
  Single-credit branch уже использовал `<button onClick={goToArtist}>`
  где `goToArtist` сам делает `closeFullscreen()` + `navigate(...)` — не
  тронут.
- На мобиле раньше клик по артисту в multi-credit случае менял URL, но
  fullscreen overlay (z-50) оставался поверх свежезагруженной artist
  page → пользователь не видел перехода. Теперь fullscreen стартует
  exit-анимацию в тот же React commit, что и navigate → плавный
  кроссфейд между фуллскрином и artist page.

**5. PWA hardening — offline-устойчивость.**

`vite.config.ts` workbox config дополнен:
- `globPatterns: ['**/*.{js,css,html,svg,png,ico,webp,woff2,woff,ttf}']` —
  явный список precache-расширений; future fonts/silent-loop assets
  попадут автоматически.
- `navigateFallback: 'index.html'` + `navigateFallbackDenylist` для
  `/api/`, `/worker/`, `/auth/`, `/ws/`. Без navigateFallback deep-links
  типа `/bratan-music/album/123` не находились в precache и валились с
  network error офлайн. Теперь SW отдаёт `index.html` → React Router
  собирает корректную страницу из cached state.
- `runtimeCaching`:
  - StaleWhileRevalidate для всех `*.png|jpg|jpeg|webp|gif|svg` (cover
    artwork) → home / library / artist / album страницы рендерятся с
    обложками после первого онлайн-визита.
  - StaleWhileRevalidate для `tidal.com` / `tidalhifi.com` доменов
    (CDN обложек) с TTL 30 дней / max 240 entries.

`src/main.tsx`:
- На boot добавлен запрос `navigator.storage.persist()` (с предварительным
  `persisted()` чтобы не дублировать, если уже granted). Persistent
  storage защищает SW precache + IDB offline-library от автоматического
  eviction при storage pressure. Браузер может отказать (gated на
  user engagement), тогда деградируем до transient — non-fatal.

Precache количество выросло 10 → 15 entries (1198 → 1482 KiB) — добавились
404.html, manifest.webmanifest, favicon.svg + extra icons.
