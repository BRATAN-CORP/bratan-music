### Что изменилось

**1. Queue modal — центрирование на любом breakpoint.**
- `src/components/features/QueueDialog.tsx`:
  - Заменён `<Sheet>` (`align="sheet"` — bottom-anchored на mobile) на
    `<Modal align="center">`. Теперь панель центрируется в viewport на
    любой ширине: центрированная модалка одинаково читается рядом с
    кнопкой queue в шапке fullscreen-плеера и в обычном UI. Скрим
    z-60 покрывает мобильный док (z-40), так что док блюрится фоном
    пока модалка открыта.
  - `panelClassName` подстроен: `max-h-[calc(100dvh-9rem-var(--pwa-safe-bottom))]`
    оставляет ~4.5rem зазора сверху и снизу под центрированную раскладку
    (mobile bottom dock + safe-bottom inset).

**2. Queue drag — split touch vs mouse.**
- Drag-to-reorder теперь идёт через `useDragControls()` +
  `Reorder.Item dragListener={false}`, как уже сделано в
  `PlaylistTrackItem`.
- На `Reorder.Item.onPointerDown`: `dragControls.start(e)` вызывается
  только если `e.pointerType === 'mouse'`. Это сохраняет старое
  поведение «драг с любого места» для мыши (на десктопе нет конфликта
  scroll-vs-drag — колесо скроллит, pointer-down таскает строку).
- На handle `<span>` (с `<GripVertical>`) `onPointerDown`:
  `e.preventDefault() + e.stopPropagation() + dragControls.start(e)`.
  preventDefault подавляет нативный text-select / image-drag, а
  stopPropagation предотвращает двойной запуск от bubble-up при mouse.
- Handle получил `touch-none`, чтобы палец на иконке шёл напрямую в
  motion-pipeline; остальная строка наследует `pan-y` от родителя
  (`data-allow-pan-y` на queue-list контейнере), так что touch
  где-нибудь, кроме иконки, скроллит список нативно.
- Добавлен i18n-ключ `queue.dragHandle` (RU «Перетащить трек» / EN
  «Drag track»), `aria-label` + `role="button"` на handle для
  доступности.

**3. Lyrics на <md — замещают обложку, кебаб → крестик.**
- `src/components/features/LyricsPanel.tsx`: внутренний компонент
  `LyricsContent` теперь экспортируется. Рендер тела (loading, error,
  empty, plain, synced + auto-scroll) не изменился.
- `src/components/layout/FullscreenPlayer.tsx`:
  - Cover-wrapper (motion.div со swipe-x навигацией, halo, TiltCard,
    AnimatePresence-crossfade) обёрнут в условный switch:
    `{lyricsOpen && !isMdUp ? <LyricsContent ...> : <coverWrapper />}`.
    Lyrics-блок: `flex-1 min-h-0 w-full max-w-md overflow-hidden`,
    `data-no-sheet-drag` чтобы не триггерить swipe-down dismiss
    fullscreen-плеера. Это и реализует «замещали обложку и пространство
    сверху чуть» — flex-1 поглощает свободное вертикальное
    пространство вокруг исходного слота обложки.
  - Удалён `<LyricsPanel mode="overlay">` рендер (был
    `absolute inset-0 z-30` поверх плеера). Mode="overlay" в самом
    `LyricsPanel` оставлен — больше не используется, но публичный
    API не ломаем (его никто другой не зовёт).
  - `MoreHorizontal` (`…`) кнопка в шапке fullscreen-плеера: при
    `lyricsOpen && !isMdUp` показывает `<X>` и закрывает лирику
    `setLyricsOpen(false)`. На md+ кнопка остаётся обычным
    кебаб-меню (там лирика — side-panel со своим dismiss). При
    замене иконки `aria-haspopup` / `aria-expanded` не выставляются
    (X — не trigger меню).
  - `LyricsContent` импортируется наряду с `LyricsPanel` из того же
    модуля.
- На md+ слой не тронут: side-panel (`LyricsPanel mode="side"`)
  по-прежнему рендерится absolute-positioned справа, cover-column
  по-прежнему сдвигается на `-22%` через
  `animate={{ x: lyricsOpen && isMdUp ? '-22%' : '0%' }}`.

## Lyrics layout follow-up: no-shift cover-slot, centred liquid-glass auto-scroll button, hidden scrollbar, swipe-to-dismiss in free area — `devin/1778410247-lyrics-no-shift`

**Кто:** Devin (по запросу пользователя после мерджа PR #429)
**Branch:** `devin/1778410247-lyrics-no-shift`

> двигать lyrics.
>

> панельки текста, и сделать чтобы она была такой же стеклянной
> (посмотри какие стили у например у navbar. это касается всех версий
> экранов. также если свайпается за облостью lyrics и плеера (только
> сччитай по свободной областю, то свайп свернуть полноэкранный плеер
> работает.
>
> также не нужно показывать скроллер

### Что изменилось

**1. Lyrics на <md — drop-in замена слота обложки, элементы плеера не двигаются.**

`src/components/layout/FullscreenPlayer.tsx`:
- `lyrics-cover-slot` div теперь имеет точно те же размерные классы,
  что и cover-wrapper: `aspect-square w-full max-w-md` + inline
  `style={{ maxWidth: 'min(28rem, calc(100vh - 28rem))' }}`. Раньше
  было `flex-1 min-h-0` — лирика поглощала свободное вертикальное
  пространство, и из-за этого title/progress/transport/volume сдвигались
  на ~½ дыхательного зазора. Теперь лирика занимает ровно тот же
  визуальный слот, что и обложка, и нижние элементы остаются на тех же
  пикселях.

**2. «Follow along» кнопка — по центру + liquid-glass.**

`src/components/features/LyricsPanel.tsx`:
- Кнопка обёрнута в `<div className="sticky bottom-4 z-10 flex
  justify-center pointer-events-none">` — теперь она inline-flex
  внутри центрирующего sticky-контейнера. Раньше использовала
  `block + left-1/2 + -translate-x-1/2`, и из-за `block` растягивалась
  на 100% ширины с лево-выровненным текстом, что визуально читалось
  как «не по центру».
- Стилизация: `liquid-glass` (тот же класс, что у navbar /
  mobile dock / popovers — multi-layer box-shadow + backdrop-filter
  blur, определён в `globals.scss`) + `rounded-full px-4 py-2 text-xs`
  + soft accent-soft halo через `shadow-[0_0_24px_-2px_var(--color-accent-soft)]`.
- Wrapper — `pointer-events-none`, кнопка — `pointer-events-auto`,
  чтобы пустое место вокруг кнопки пропускало wheel/touch-события
  на список строк под ней.
- Применяется и на `mode="side"` (md+), и на `mode="cover-slot"` (<md),
  и на оставшийся `mode="overlay"` API — везде использует один и тот
  же `LyricsContent`.

**3. Скроллбар лирики — скрыт.**

`src/styles/globals.scss`:
- Добавлен утилитарный класс `.no-scrollbar` (`scrollbar-width: none`
  + `-ms-overflow-style: none` + `::-webkit-scrollbar { display: none }`).
- `LyricsContent` контейнер теперь имеет `no-scrollbar` рядом с
  `overflow-y-auto`. Скролл по wheel / touch / клавиатуре работает
  как раньше; визуально полоса прокрутки исчезает.

**4. Swipe-to-dismiss fullscreen — работает при открытой лирике.**

`src/components/layout/FullscreenPlayer.tsx`:
- `lyricsOpen` убран из массива `anyModalOpen` (раньше блокировал
  swipe-to-dismiss целиком, пока лирика была открыта). Это было
  оправдано во времена `mode="overlay"` (лирика покрывала весь плеер
  как отдельное окно), но теперь лирика — либо replacement в слоте
  обложки на <md, либо absolute-side-panel на md+; ни в том, ни в
  другом случае она не является полноэкранным modal-ом.
- Лирика-контейнер уже несёт `data-no-sheet-drag`, что соответствует
  `SHEET_DRAG_EXCLUDE_SELECTOR` в `startSheetDrag`. Поэтому pointer-down
  В пределах лирики по-прежнему НЕ запускает drag (wheel/touch скролл
  лирики работает корректно), но pointer-down в свободной зоне
  плеера (выше лирики, по бокам, между transport и volume) теперь
  снова стартует sheet-drag и сворачивает фуллскрин.
- Контракт: `eqOpen / queueOpen / addToPlaylistOpen / overrideOpen /
  moreOpen` остаются в `anyModalOpen` — это настоящие модальные
  окна со своими scrim'ами и dismiss-affordance.

- Audio engine, security, миграции — не тронуты.
- `mode="side"` lyrics на md+ — не тронут (только `LyricsContent`
