### Что изменилось

**1. Queue modal — центрирование на любом breakpoint.**
- `src/components/features/QueueDialog.tsx`:
  - Заменён `<Sheet>` (`align="sheet"` — bottom-anchored на mobile) на
    `<Modal align="center">`. Теперь панель центрируется в viewport на
    любой ширине: центрированная модалка одинаково читается рядом с
    кнопкой queue в шапке fullscreen-плеера и в обычном UI. Скрим
    z-60 покрывает мобильный док (z-40), так что док блюрится фоном
    пока модалка открыта.
  - `panelClassName` подстроен: `max-h-[calc(100dvh-9rem-var(--pwa-safe-bottom))]`
    оставляет ~4.5rem зазора сверху и снизу под центрированную раскладку
    (mobile bottom dock + safe-bottom inset).

**2. Queue drag — split touch vs mouse.**
- Drag-to-reorder теперь идёт через `useDragControls()` +
  `Reorder.Item dragListener={false}`, как уже сделано в
  `PlaylistTrackItem`.
- На `Reorder.Item.onPointerDown`: `dragControls.start(e)` вызывается
  только если `e.pointerType === 'mouse'`. Это сохраняет старое
  поведение «драг с любого места» для мыши (на десктопе нет конфликта
  scroll-vs-drag — колесо скроллит, pointer-down таскает строку).
- На handle `<span>` (с `<GripVertical>`) `onPointerDown`:
  `e.preventDefault() + e.stopPropagation() + dragControls.start(e)`.
  preventDefault подавляет нативный text-select / image-drag, а
  stopPropagation предотвращает двойной запуск от bubble-up при mouse.
- Handle получил `touch-none`, чтобы палец на иконке шёл напрямую в
  motion-pipeline; остальная строка наследует `pan-y` от родителя
  (`data-allow-pan-y` на queue-list контейнере), так что touch
  где-нибудь, кроме иконки, скроллит список нативно.
- Добавлен i18n-ключ `queue.dragHandle` (RU «Перетащить трек» / EN
  «Drag track»), `aria-label` + `role="button"` на handle для
  доступности.

**3. Lyrics на <md — замещают обложку, кебаб → крестик.**
- `src/components/features/LyricsPanel.tsx`: внутренний компонент
  `LyricsContent` теперь экспортируется. Рендер тела (loading, error,
  empty, plain, synced + auto-scroll) не изменился.
- `src/components/layout/FullscreenPlayer.tsx`:
  - Cover-wrapper (motion.div со swipe-x навигацией, halo, TiltCard,
    AnimatePresence-crossfade) обёрнут в условный switch:
    `{lyricsOpen && !isMdUp ? <LyricsContent ...> : <coverWrapper />}`.
    Lyrics-блок: `flex-1 min-h-0 w-full max-w-md overflow-hidden`,
    `data-no-sheet-drag` чтобы не триггерить swipe-down dismiss
    fullscreen-плеера. Это и реализует «замещали обложку и пространство
    сверху чуть» — flex-1 поглощает свободное вертикальное
    пространство вокруг исходного слота обложки.
  - Удалён `<LyricsPanel mode="overlay">` рендер (был
    `absolute inset-0 z-30` поверх плеера). Mode="overlay" в самом
    `LyricsPanel` оставлен — больше не используется, но публичный
    API не ломаем (его никто другой не зовёт).
  - `MoreHorizontal` (`…`) кнопка в шапке fullscreen-плеера: при
    `lyricsOpen && !isMdUp` показывает `<X>` и закрывает лирику
    `setLyricsOpen(false)`. На md+ кнопка остаётся обычным
    кебаб-меню (там лирика — side-panel со своим dismiss). При
    замене иконки `aria-haspopup` / `aria-expanded` не выставляются
    (X — не trigger меню).
  - `LyricsContent` импортируется наряду с `LyricsPanel` из того же
    модуля.
- На md+ слой не тронут: side-panel (`LyricsPanel mode="side"`)
  по-прежнему рендерится absolute-positioned справа, cover-column
  по-прежнему сдвигается на `-22%` через
  `animate={{ x: lyricsOpen && isMdUp ? '-22%' : '0%' }}`.
