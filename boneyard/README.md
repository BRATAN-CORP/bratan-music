# Boneyard

Кладбище для **снятого с продакшена, но не удалённого** кода BRATAN Music.

Сюда переезжает то, что:

- больше не импортируется из `src/`, `worker/src/`, `bot/src/`,
- но **может пригодиться** как референс при отладке регрессий, A/B-сравнении
  старого и нового UX, восстановлении удалённой фичи или при разборе
  легаси-багов.

Когда такая необходимость отпадает — модуль удаляется из репозитория
обычным `git rm`. До этого момента он лежит здесь, чтобы будущим агентам
не приходилось копаться в `git log` ради двухсотстрочного компонента,
который уже никто не помнит.

## Как класть код в boneyard

1. Перенеси файл в `boneyard/<категория>/<имя>.tsx` (например
   `boneyard/components/OldExplorePage.tsx`). Сохрани оригинальные пути
   в комментарии-шапке файла, чтобы было видно, откуда он пришёл:

   ```ts
   // Originally lived at src/app/explore/page.tsx until PR #NNN
   // (replaced by the unified search-page experience).
   ```

2. Удали оригинал из `src/` и проверь, что репо собирается без него:

   ```bash
   npm run lint
   npm run typecheck
   npm run build
   ```

3. **Не правь** код после переноса. Boneyard — read-only архив. Если
   нужно восстановить компонент — скопируй его обратно в `src/` и
   правь там.

## Что НЕ кладём в boneyard

- Секреты, токены, ключи, дампы БД — стандартные правила репозитория
  применяются и здесь.
- Аудио-движок (`useAudioPlayer`, `Player.tsx`, `FullscreenPlayer.tsx`,
  визуализатор / EQ / lyrics). Эти модули защищены hard-constraints в
  [`AGENTS.md`](../AGENTS.md) — их историю смотрим через `git log`, а не
  через копию в boneyard.
- Применённые D1-миграции (`worker/migrations/*.sql`). Они — часть
  схемы, и копия в boneyard введёт в заблуждение.

## Сборка

Boneyard **исключён** из:

- TypeScript (`tsconfig.json` `include: ["src"]` — boneyard вне);
- ESLint (`eslint.config.mjs` `ignores: [..., 'boneyard/']`);
- Vite-бандла (Vite собирает только то, что импортируется из `src/`).

То есть код в boneyard физически не попадает в продакшен-бандл и не
ломает CI. Можно класть даже сломанный TypeScript — он не будет
проверен (но всё равно постарайся переносить рабочие версии — это и
есть смысл архива).
